# Project_Codeship_web
# 🚀 CodeShip  
유아 및 초등생을 위한 단계별 코딩 교육 커리큘럼 웹 플랫폼
🏆 웹 경진대회 3등 수상 | 팀장 | UI·구조·기획 총괄


연령별 발달 단계에 맞춘 코딩 커리큘럼(언플러그드 → 블록코딩 → 로봇코딩)을 제공하고,  
공지·커뮤니티·마켓·문의 기능까지 포함한 **코딩 교육 통합 웹사이트** 프로젝트입니다.

---

## 1. 프로젝트 개요

### 목표

- 유아·초등 대상 학습 난이도 차이를 반영한 **단계별 코딩 커리큘럼 제공**
  - 유아 과정: 코딩 개념 소개 → 언플러그드 컴퓨팅 → Entry/Scratch
  - 초등 과정: 언플러그드 심화 → Scratch → M Bot(로봇 코딩)
- 어린이 친화적인 UI/UX를 통해 **학습 흐름을 직관적으로 전달**
- 웹 경진대회 출품을 목표로,  
  단순 소개 페이지가 아닌 **실제 서비스 형태의 웹 플랫폼 구현**

### 핵심 접근 방식

- 콘텐츠 나열이 아닌 **학습 로드맵(Progressive Roadmap)** 중심 설계
- Bootstrap 기반 반응형 웹으로 모바일·태블릿·PC 환경 대응
- 기존 템플릿(KidKinder)을 그대로 사용하지 않고  
  **브랜딩·정보 구조·콘텐츠를 전면 재설계**

---

## 2. 전체 아키텍처 개요

1. 입력: 사용자 웹 브라우저 접속
2. 메인 페이지 (`index.html`)
   - 플랫폼 소개 및 주요 기능 진입
3. 커리큘럼 페이지 (`elementary.html`)
   - 유아/초등 과정 분리
   - Step 기반 학습 흐름 제공
4. 강의 목록 페이지 (`elementary_lectures.html`)
   - Step별 강의 및 외부 학습 자료(YouTube 등) 연결
5. 부가 기능 페이지
   - 공지사항: `notice.html` (Sticky Notes UI)
   - 커뮤니티 UI
   - 마켓: `market.html`
   - 문의하기: `contact.html`
6. 공통 UI/UX
   - Bootstrap Grid 기반 레이아웃
   - jQuery 기반 UI 인터랙션
   - Owl Carousel / Lightbox 등 플러그인 활용

---

## 3. 주요 기능 상세

### 3.1 단계별 코딩 커리큘럼

#### Kinder Coding Step (유아 과정)
- Step 1: 코딩이란?
- Step 2: 언플러그드 컴퓨팅
- Step 3: Entry & Scratch

#### Elementary Coding Step (초등 과정)
- Step 1: 언플러그드 컴퓨팅 심화
- Step 2: Scratch 블록 코딩
- Step 3: M Bot 로봇 코딩

→ 연령별 인지 수준을 고려해 **난이도가 점진적으로 상승**하도록 설계

---

### 3.2 공지사항 (Sticky Notes UI)

- 게시판 형태가 아닌 Sticky Notes 컨셉 UI 적용
- 어린이 사용자도 쉽게 공지 내용을 인지할 수 있도록 시각적 강조

---

### 3.3 커뮤니티 / 마켓 / 문의 기능

- 커뮤니티: 사용자 간 소통을 고려한 게시 UI 구조
- 마켓: 교육 관련 콘텐츠/상품 확장을 고려한 페이지
- 문의하기: Contact Form 기반 사용자 문의 접수

---

## 4. 개발 파이프라인 및 설계 포인트

### 4.1 UI/UX 설계

- 어린이 대상 서비스 특성을 반영
  - 텍스트 최소화
  - 아이콘·이미지 중심 정보 전달
  - 단순한 페이지 이동 구조

### 4.2 반응형 웹 적용

- Bootstrap Grid System 활용
- 모바일·태블릿·PC 환경에서 일관된 사용자 경험 제공

### 4.3 템플릿 커스터마이징

- 원본 템플릿: **KidKinder (HTML Codex)**
- 변경 사항
  - 브랜드명: KidKinder → CodeShip
  - 아이콘 컨셉: Teddy Bear → Coding/Terminal
  - 콘텐츠 구조: 유아·초등 코딩 교육 특화 구조로 재설계
  - 한글/영어 혼합 인터페이스 구성

## 6. 주요 시도 및 실험

1. 커리큘럼을 **단계형 학습 로드맵**으로 재구성  
   - 연령별 학습 흐름이 자연스럽게 이어지도록 설계
2. 템플릿 기반 개발의 한계 극복  
   - 디자인·정보 구조·콘텐츠를 프로젝트 목적에 맞게 재설계
3. 공지사항 UI 차별화  
   - 기존 게시판 대신 Sticky Notes UI로 접근성 및 흥미 요소 강화
4. 확장 가능한 구조 설계  
   - 추후 로그인, 학습 진도 관리, 온라인 코딩 에디터 연동을 고려한 페이지 분리
